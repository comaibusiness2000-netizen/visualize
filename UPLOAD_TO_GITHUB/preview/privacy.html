Visualize privacy policy draft.

This draft describes the production privacy model planned for the App Store version. It is not legal advice and should be reviewed before launch.

Prototype status

The current preview saves data locally on the device/browser. The production app is designed to use account-based cloud sync, secure image storage, and server-side AI generation.

Data we expect to store

  • Account identifier from Sign in with Apple.
  • Goals, daily tasks, short-term goals, Vision decks, Anti-vision decks, and audio script settings.
  • Photos uploaded by the user and AI-generated or AI-edited images.
  • AI prompts needed to generate or edit images and scripts.
  • Subscription entitlement status from Apple In-App Purchase.

How data should be used

User content should be used only to provide visualization, anti-vision, audio, AI generation, sync, account recovery, and subscription features.

Storage and sync

Images should be compressed before upload and stored in private object storage. Database rows should store metadata and storage references, not raw image data.

AI processing

AI generation and image editing should run on the server, not directly from the app. API keys must never be embedded in the client.

Deletion

Production users should be able to delete their account, goals, decks, images, prompts, generated content, and audio scripts.

Payments

Subscriptions should use Apple In-App Purchase. The backend should validate receipts/server notifications and store entitlement state.
